THE CASE IN 30 SECONDS

Situation: Home Depot built great AI-powered products (chatbot, recommendations, search, ads). Each works well alone.

Problem: The products don't coordinate. Customers get conflicting advice, lose trust, defect to Lowe's.

Constraint: CEO forbids centralization or bureaucracy. Must preserve speed and autonomy.

Ask: Design an AI-powered solution that coordinates products WITHOUT slowing teams down.

Core Insight: Every team optimizes for their own proxy metric, but all metrics are downstream of the same thing: accuracy to reality.

THE PROBLEM - WHAT'S ACTUALLY BREAKING

+

The One-Line Version

Each team optimizes their own metrics, but nobody owns the customer journey.

What "Capability Collision" Looks Like

→
Chatbot recommends Product A

→
Sponsored Ad pushes Product C
→
Customer confused, leaves

The Four Root Causes

1. Org Structure

Vertical teams own products. Nobody owns the horizontal customer journey where products intersect.

2. Incentive Misalignment

Each team measured on their own OKRs. Optimizing YOUR metric can hurt THEIR metric.

3. Data Fragmentation

Different data sources, no real-time sync, AI models trained separately with different assumptions.

4. Governance Gap

No process for resolving conflicts between teams. No one decides who "wins" when products clash.

"We've optimized for product autonomy so much that we've lost system harmony. We've given everyone a compass, but no one is using the same map."
— Sam, EVP Digital Products

DAVE'S THREE COLLISIONS FRAMEWORK

+

Dave, VP of Delivery, identified three distinct collision patterns:

1. Data Collision

Teams using different or stale data sources. Chatbot says "in stock" while store system says "sold out."

2. Logic Collision

Conflicting business rules across products. Recs optimizes for margin while Ads optimizes for revenue share.

3. Experience Collision

Customer sees inconsistent information across touchpoints. Three different price points for the same item.

SOLUTION DIRECTION - THE VISIBILITY LAYER

+

The Core Insight

AI as Fragmentation Amplifier

Each team deploys AI independently. Different data, different models, different optimization targets. AI makes collisions faster, harder to detect, harder to fix.

AI as Coordination Layer

Use AI to monitor interactions between products in real-time. Detect conflicts before customers see them. Transform static metrics into dynamic coordination.

"AI does not necessarily create new redundancy, it multiplies the speed, scale, and impact of redundancy."
— Sam, EVP Digital Products

The Visibility Layer Concept

A lightweight event stream where teams publish their customer-facing decisions, enabling real-time conflict detection without centralizing decision-making power.

It IS It ISN'T
Event stream (ephemeral) Data warehouse (permanent)
Conflict detection Conflict resolution
Shared visibility Centralized control
Additive layer Replacement of existing systems
Schema-light Complex ETL pipelines

What the Layer Does

1. Detect Monitor when AI models from different teams make conflicting decisions about the same customer/product
2. Alert Notify teams before conflicts reach customers (not after complaints come in)
3. Resolve When signals conflict, intelligently decide which one serves customer intent best
4. Learn Track patterns, identify recurring collision points, improve over time

The Shared OKR: "Accuracy to Reality"

All teams adopt a new shared metric alongside their existing OKRs:

  • Conflict rate — lower = better
  • Resolution time — faster = better
  • Customer impact incidents — fewer = better

From Static to Dynamic

Current (Static) Proposed (Dynamic)
Redundancy measured quarterly Coherence measured in real-time
Problems found via customer complaints Problems detected before customers see them
Manual governance reviews Automated conflict detection
Teams don't know they're colliding Teams alerted instantly when interactions conflict

WHAT WE NEED TO DELIVER

+

The Four Required Sections

  • 1. Root Cause Analysis
    WHY are capabilities colliding? Go beyond symptoms to underlying org, tech, and governance factors.
  • 2. Strategic Recommendation
    HOW do we fix it? Must include AI transforming redundancy from static to dynamic. Specific governance changes, shared metrics, coordination mechanisms.
  • 3. Financial Analysis
    Cost-benefit model. Investment required, expected return, cost of doing nothing.
  • 4. Implementation Roadmap
    Phased approach: Magic Apron pilot first, then scale. Milestones, success metrics, decision gates.

Implementation Roadmap

Phase 1: Instrument (Weeks 1-4) Define event schema. Magic Apron + Inventory publish to stream. Basic conflict detection rules.
Phase 2: Visibility (Weeks 5-8) Dashboard for cross-team visibility. Alert system for threshold breaches. Shared OKR tracking.
Phase 3: AI Analysis (Weeks 9-12) Batch processing for pattern detection. Process mining on historical events. Automated escalation.
Phase 4: Expand (Months 4-6) Add Recs/Search to event stream. Add Sponsored Ads. Refine conflict rules based on learnings.
